# ADXL345 实时传感数据管线项目技术说明与论文参考（中文版）

## 1. 项目概述

### 1.1 项目名称

基于 Raspberry Pi 5、ADXL345、MQTT 与 Qt 的实时传感数据采集与可视化系统。

### 1.2 项目目标

本项目实现一条完整的数据链路：

1. 树莓派端通过 I2C 读取 ADXL345 三轴加速度。
2. 树莓派端计算姿态角（Pitch/Roll）并采集 CPU 运行信息。
3. 通过 MQTT 将 JSON 数据实时发布到 Broker。
4. Windows Qt 上位机订阅并解析数据，进行实时曲线绘制与滚动统计。
5. 支持运行状态主题（LWT）与用户友好的中英文切换界面。

## 2. 系统总体架构

## 2.1 架构分层

系统由三层组成：

1. 感知层：ADXL345 传感器（I2C）。
2. 网络传输层：MQTT Broker（Mosquitto）。
3. 应用层：Qt 桌面可视化上位机 + 树莓派业务程序。

## 2.2 数据流说明

1. ADXL345 输出原始加速度寄存器数据（X/Y/Z）。
2. 树莓派端程序读取并计算 Pitch、Roll。
3. 树莓派端将姿态与系统信息封装为 JSON，发布到 MQTT 主题。
4. Qt 上位机订阅数据主题和设备状态主题，展示图形与统计结果。

## 4. 核心技术原理

### 4.1 姿态角计算

项目中 Pitch/Roll 来自加速度重力分量近似计算：

- Pitch 基于 X 与 (Y,Z) 分量关系。
- Roll 基于 Y 与 (X,Z) 分量关系。

实现位于 ADXL345.cpp 的 calculatePitchAndRoll()。

### 4.2 MQTT 通信机制

1. 发布端：publisher.cpp，默认每 2 秒发布一次 JSON。
2. 订阅端：Qt 上位机与 subscriber_led_wiringpi.cpp。
3. 状态主题（LWT）：sensor/adxl345/status。
   - 上线发布 online。
   - 异常断开由遗嘱消息发布 offline。

### 4.3 Qt 上位机处理逻辑

1. 连接参数配置：Broker、ClientID、用户名密码、QoS。
2. 多主题订阅：数据主题 + LWT 状态主题。
3. 数据处理：JSON 解析、字段容错映射、时间戳提取。
4. 可视化：实时曲线、最近 60 秒统计（均值/最小/最大）。
5. 交互优化：中英文切换、连接引导、缓存清空。

## 5. MQTT 主题与数据协议

### 5.1 默认主题

1. 数据主题：sensor/adxl345
2. 状态主题：sensor/adxl345/status

注意：subscriber_led_wiringpi.cpp 当前默认订阅主题为 sensor/adxl345/PC，如需接收 publisher.cpp 的数据，请统一主题配置。

### 5.2 JSON 字段定义

推荐发布格式：

```json
{
  "pitch": -31.88,
  "roll": -52.19,
  "accel_x": -129,
  "accel_y": -193,
  "accel_z": -76,
  "cpu_load": 0.12,
  "cpu_temp_c": 51.8,
  "timestamp": "2026-04-10T07:17:46Z"
}
```

字段说明：

1. pitch/roll：姿态角，单位 deg。
2. accel_x/accel_y/accel_z：三轴加速度原始值。
3. cpu_load：系统负载。
4. cpu_temp_c：CPU 温度（摄氏度）。
5. timestamp：UTC 时间戳（ISO8601）。

## 6. 复现实验步骤（新手版）

### 6.1 第一步：准备 MQTT Broker

Windows 建议安装 Mosquitto，并确认服务已启动：

```powershell
Get-Service mosquitto | Select-Object Name, Status, StartType
Start-Service mosquitto
Get-NetTCPConnection -LocalPort 1883 -State Listen
```

### 6.2 第二步：启动 Qt 上位机

在项目根目录执行：

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\build_qt.ps1
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\run_qt.ps1
```

### 6.3 第三步：运行树莓派发布端

在 Raspberry Pi 上编译并运行 publisher（示例命令，库路径按本机调整）：

```bash
g++ -std=c++17 publisher.cpp ADXL345.cpp I2CDevice.cpp -ljsoncpp -lpaho-mqtt3c -o publisher
./publisher 1
```

说明：

1. 程序参数 0/1/2 表示 MQTT QoS。
2. 运行后每 2 秒向数据主题发布一次 JSON。

### 6.4 第四步：观察结果

1. Qt 上位机日志出现实时数据。
2. 曲线随传感器姿态变化而更新。
3. 统计区域显示最近 60 秒均值/最小/最大值。
4. 状态主题 online/offline 能反映设备在线状态。

## 8. 常见问题与排查

### 8.1 Qt 点击连接卡住

1. 若是本机 Broker，先确认 Mosquitto 服务启动。
2. 当前版本已增加服务状态检查与连接超时，卡顿已明显降低。

### 8.2 能连接但没有曲线

1. 检查发布主题与订阅主题是否一致。
2. 检查当前选择指标是否在 JSON 中存在。

### 8.3 树莓派端传感器读取失败

1. 检查 I2C 是否启用。
2. 检查 ADXL345 接线与地址（默认 0x53）。
3. 检查执行权限（必要时 sudo 运行）。

### 8.4 LED 程序无反应

1. 检查 wiringPi 是否安装。
2. 检查 GPIO 引脚映射是否与代码一致。
3. 检查 subscriber 的订阅主题是否与发布主题一致。
