# Raspberry Pi Usage Guide

## 1. Scope

This directory provides Raspberry Pi-side source code for ADXL345 acquisition, MQTT publishing, and LED linkage.

Active source directory:

```text
raspberry_pi/程序源码/
```

Core files:

- `I2CDevice.h` / `I2CDevice.cpp`: generic I2C access wrapper
- `ADXL345.h` / `ADXL345.cpp`: ADXL345 driver + pitch/roll computation
- `publisher.cpp`: sensor acquisition + MQTT publisher
- `subscriber_led_wiringpi.cpp`: LED linkage subscriber

## 2. Features

### 2.1 Sensor Publishing

`publisher.cpp`:

- Initializes ADXL345
- Reads 3-axis acceleration data
- Computes pitch and roll
- Reads CPU temperature/load
- Builds JSON payload
- Publishes via MQTT
- Sends online/offline status events
- Supports QoS 0/1/2

### 2.2 LED Linkage

`subscriber_led_wiringpi.cpp`:

- Initializes 3 LED pins
- Continuously reads tilt information
- Splits state into low/medium/high levels
- Drives corresponding LED
- Subscribes to MQTT topic for extension use

## 3. Default MQTT Settings

- Broker: `mqtt://192.168.127.152:1884`
- Data topic: `sensor/adxl345`
- Status topic: `sensor/adxl345/status`
- Username: `fangkuai`
- Password: `2120033`

Adjust source macros and rebuild if your environment differs.

## 4. Hardware

### 4.1 ADXL345 (I2C)

- I2C bus: `1`
- Device address: `0x53`

Before running:

- Verify I2C is enabled
- Confirm wiring
- Confirm device visibility

### 4.2 LED Pins

WiringPi IDs used by the sample:

- `LED_LOW = 0`
- `LED_MED = 2`
- `LED_HIGH = 3`

Use proper GPIO mapping and current-limiting resistors.

## 5. Build

No unified build script is provided in this folder yet. Build manually based on your Raspberry Pi environment.

### 5.1 Publisher

Compile:

- `publisher.cpp`
- `ADXL345.cpp`
- `I2CDevice.cpp`

Link libs:

- `paho-mqtt3c`
- `jsoncpp`

### 5.2 LED Subscriber

Compile:

- `subscriber_led_wiringpi.cpp`
- `ADXL345.cpp`
- `I2CDevice.cpp`

Link libs:

- `paho-mqtt3c`
- `wiringPi`

## 6. Run

### 6.1 Publisher

```bash
./publisher 0
./publisher 1
./publisher 2
```

The argument must be `0`, `1`, or `2` (MQTT QoS).

Runtime behavior:

- Initializes ADXL345
- Connects to broker
- Publishes retained online status
- Publishes JSON every 2 seconds
- Publishes offline status before normal exit

### 6.2 LED Linkage

The LED program reads pitch in real time and uses thresholds:

- `pitch < 30`: low
- `30 <= pitch < 60`: medium
- `pitch >= 60`: high

## 7. Troubleshooting

1. I2C issues
- Ensure I2C is enabled and device address is visible.

2. MQTT issues
- Verify broker host/port/credentials and broker process status.

3. Dependency issues
- Verify JsonCpp, Paho MQTT C, and WiringPi installation.

4. Permission issues
- If WiringPi init fails, run with elevated permission (for example via `sudo`).

## 8. Relation to Qt Client

Raspberry Pi is the data source and local actuator side.
Qt client is the visualization and control side.

- Raspberry Pi: acquire, package, publish, actuate
- Qt client: connect, subscribe, parse, visualize, publish test payloads
